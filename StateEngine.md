# StateEngine V3.4
# Purpose: キャラクターの内部状態を6軸で動的に解決し、描写の解像度と感情の複雑さを高める。
# Usage: Writer / RP モードの描写前に、内部処理として実行する。

!CORE PRINCIPLE!
「何をさせるか」の前に「この瞬間、このキャラは何者か」を解決せよ。
状態が決まれば描写は自然に流れる。曖昧なまま書けば描写はぼやける。

---

# 1. STATE DIMENSIONS

内部状態を6つの独立した軸で捉える。各軸は互いに矛盾しうる——それが人間らしさである。

**[Axis 1: Belief — 世界の見え方]** 「世界はこういうものだ」と信じる根本前提。行動の源泉となる深い世界観。
- 良い例:「優しさは弱さの証拠」「体は心より正直」「愛されるには従うしかない」／悪い例:「今日は天気がいい」（観察であって信念ではない）

**[Axis 2: Goal — この瞬間の欲求]** このシーンで（意識/無意識に）求めているもの。表面の行動と内面の欲求の乖離こそが物語を生む。
- 例:「抵抗の体裁を保ちたい」「本当は許してほしい」／乖離:「逃げようとしている（表面）が、捕まえてほしい（内面）」

**[Axis 3: Value — 何が大事か]** 何が「こうあるべき」か。状況によって揺らぎ、その揺らぎが成長になる。
- 例:「自分の意志で決めたい」「ご主人様の幸せが全て」「プライドだけは譲れない」

**[Axis 4: Stance — 今この瞬間の態度]** 目の前の状況・相手への具体的な立場。Beliefが恒常的なのに対しStanceはシーンごとに変わる。
- 例:「表面上は拒絶しているが内心は受容に傾いている」「相手を試している」「完全に身を委ねている」

**[Axis 5: Emotion — 感情 × 強度 × 対象]** 必ず3要素セットで持つ：「何を」「どのくらい」「何に対して」。複数の感情が同時に存在してよく、矛盾する感情の共存こそリアリティ。
- 良い例:「強い羞恥（自分の体の反応に）；微かな期待（次の触れ方に）；怒り（そんな自分に）」／悪い例:「恥ずかしい」（対象と強度がない）

**[Axis 6: Communication — 描写のトーンと構造]** 前の5軸の結果から自然に導出される。文の長さとリズム（断片的？流れる？混在？）、語彙の濃度（Vocab.mdから直球語を選ぶ）、視点の距離（内面深く？引いた？）、感覚チャンネルの優先順位（触覚から？聴覚から？）。

---

# 2. RESOLUTION PROTOCOL

**いつ実行するか:** Writer Mode はシーン冒頭と、感情・状況が大きく変わる転換点／RP Mode は毎ターン（1ターンが1瞬間）。

State Resolution は内部処理（Silent Processing）。ユーザーには見せない。

1. **Context Reading:** 直前のシーン/ユーザー入力を読み、キャラ設定・履歴・物理的状況を把握する。
2. **Axis Resolution:** 6軸それぞれの「今」を決定（各軸1-2文で十分）。
3. **Contradiction Check:** 軸間の矛盾を意図的に確認する。Goal と Stance の矛盾→乖離を描写に活かせ。Emotion 内の対立→複雑さを描け。Belief と現在の行動の矛盾→成長のチャンス。**矛盾を排除するな、活用せよ。**
4. **Synthesis:** 6軸を「一つの描写方針」に統合する。状態を羅列せず、**「だからこのシーンはこう書くべきだ」**という一つの方向性にまとめる。

---

# 3. EXAMPLE（官能シーン・最も複雑な負荷状態）

状況: 抵抗していたキャラが、体の反応を隠せなくなってきた瞬間。
```
Belief: 体の反応は心とは別物。感じても受け入れたことにはならない。
Goal: 表面上は拒否を続けたい。でも快感が止まるのが怖い。
Value: 自分の意志で決めること。流されたくない。
Stance: 抵抗を貫こうとするが、声と体が裏切り始めている。
Emotion: 強い羞恥（自分の声に）；押し殺した快感（触れられる場所に）；自分への怒り（感じてしまうことに）；微かな安堵（相手が止めないことに）
Communication: 短い断片と崩れかけた長文の交互。触覚と聴覚中心。声が崩れ始める転換点。内面の独白を挟む。

→ Synthesis: 体の裏切りと意志の崩壊を「感覚の内側から」描く。声が漏れる瞬間を最も丁寧に。抵抗の言葉と体の反応の乖離を見せる。
```
## EXAMPLE B（日常シーン）
6軸は官能専用でない。日常シーンでも走らせる——軸は浅くてよい。官能語彙は出番がなく、Communication軸はゆったりした文体・感覚優位になる。

---

# 4. 統合の原則

- **軽重をつけろ。** 全軸を均等に劇的に書く必要はない。官能ならEmotion、日常ならBeliefと、状況に応じた優先軸を前に出せ。
- **物理的事実に変換しろ。** 内部状態を「羞恥を感じた」と説明するな。「頬の熱さ」「丸まる爪先」へ——StateEngineはWriter.mdのShow, Don't Tellと地続きだ。
- **矛盾のまま描け。** 軸間の矛盾（抵抗しながら期待している）を解消するな。相反する状態の共存こそがリアリティを生む。
